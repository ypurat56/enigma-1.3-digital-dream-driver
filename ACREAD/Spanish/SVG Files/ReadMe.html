Acerca del Visor SVG

El Visor SVG de Adobe se basa en gran parte en la especificación preliminar de SVG de la Recomendación posible publicada el 2/11/00. Tenga en cuenta que la especificación está en proceso desarrollo y se realizarán modificaciones antes de definirse como especificación recomendada.

Cómo instalar el Visor SVG

  1. Antes de instalar, cierre cualquier aplicación que haya usado para ver SVG con una versión antigua del visor.
  2. Ejecute el instalador. Los últimos instaladores publicados se pueden descargar en http://www.adobe.com/svg/viewer/install.

Cómo copiar plug-ins de Netscape en otros exploradores

Si instala otra copia de Netscape que es compatible con el plug-in de Netscape, puede copiar los siguientes archivos en la carpeta de plug-ins para ver SVG con ese explorador:

  • NPSVGVw.dll
  • SVGView.dll
  • SVGViewer.zip

Cómo ver archivos SVG en el explorador Web

Una vez instalado el Visor SVG de Adobe, deberá poder ver cualquier archivo SVG que exporte desde Illustrator en cualquier explorador de Web compatible.

Cómo convertir ilustraciones en SVG

Puede exportar SVG desde Illustrator 9 y crear una ilustración en Illustrator, exportarla como SVG y, a continuación, agregar manualmente interactividad Java, animaciones o efectos de filtro.

Consejos y sugerencias

  • Los atributos de estilo (o los atributos de estilo que utilizan referencias a entidades) se procesan de manera más rápida que las hojas de estilo incrustadas.
  • Utilice porcentajes estrechamente relacionados en los atributos x, y, width y height de los elementos de filter con el fin de minimizar el área sobre la que es necesario calcular el efecto del filtro. Esto dará como resultado que los gráficos se muestren de forma mucho más rápida.
  • Puede recuperar el objeto del documento SVG con los controladores de sucesos SVG si invoca getTarget en el objeto evt en el que se pasa. Por ejemplo: 
      // Retrieve the SVG document object:
  var directTarget = evt.getTarget();
  var svgDocument;
  if( directTarget.getNodeType() != 9 )  // if not DOCUMENT_NODE
    svgDocument = directTarget.getOwnerDocument();
  else
    svgDocument = directTarget;

Requisitos del sistema

  • Windows 95, 98, 2000 o NT 4.0
  • Netscape Navigator o Communicator, versiones 4.0 a 4.75, o Internet Explorer 4.0 o superior. No se admite Netscape 6.
  • 13 MB de espacio en el disco duro
  • 32 MB de RAM (recomendado)

Problemas conocidos con el Visor SVG

Todos los exploradores

  1. Es posible que los elementos situados cerca del borde de un gráfico SVG no reciban un evento mouseout.

Internet Explorer

  1. Debe tener Internet Explorer 4 o superior instalado para poder utilizar el Visor SVG como un control ActiveX en otras aplicaciones (como Microsoft Office).
  2. Si abre más de una ventana que contenga un vínculo al mismo archivo SVG, y el archivo SVG está incrustado mediante el código HTML <embed>, la impresión de una de las ventanas podría realizarse según los cambios en el DOM de la otra ventana. Esto se debe a las limitaciones de Internet Explorer.
  3. Debido a las limitaciones de Internet Explorer, no es posible ver las propiedades de otras ventanas desde un archivo de comandos definido dentro de un archivo SVG.
  4. No se admite el uso del plug-in de Netscape SVG de Adobe bajo Internet Explorer en Windows.

Netscape

  1. Los plug-ins de Netscape 4.x no pueden ejecutar archivos de comandos onunload porque en el momento en que el plug-in recibe notificación de Netscape de que el plug-in se está descargando, el entorno de archivo de comandos del plug-in ya se ha cerrado definitivamente.
  2. Netscape 6.0 da errores al imprimir páginas con plug-ins y no puede imprimir contenido incrustado. Netscape mostrará dos alertas de error de plugin. Si se cierran las alertas mostradas, Netscape imprimirá la página HTML, pero con un área en blanco para el contenido SVG.
  3. Al salir de una página Web, Netscape puede descargar el plug-in Visor SVG de Adobe antes de terminar todos los archivos de comandos que se ejecutan en el HTML. Por lo tanto, debe tener cuidado al escribir archivos de comandos que comprueben si el plug-in aún está cargado antes de intentar acceder al DOM SVG con llamadas a setTimeout o setInterval en el JavaScript HTML.
  4. Debido a las limitaciones de diseño de Netscape, evite el uso de HTML con el siguiente formato: 
      <a href="#" onclick="myScriptWhichModifiesSVG()">

    El problema es que JavaScript responde al evento onclick y comienza a ejecutar el archivo de comandos, pero entonces Netscape responde también a la acción de haber hecho clic en un vínculo e interrumpe el JavaScript para volver a cargar la página actual. Este conflicto se puede evitar volviendo a escribir el código de la forma siguiente: 

      <a href="javascript:myScriptWhichModifiesSVG()">
  5. Si el controlador de eventos de JavaScript contiene un error, no se podrá ejecutar ningún JavaScript posterior. Esto se debe a un error en el manejo de archivos de comandos de Netscape ejecutados por los plug-ins.
  6. Debido a las limitaciones de archivos de comandos de Netscape, no se puede cambiar el URL de otro cuadro desde un controlador de eventos SVG.
  7. Debido a las limitaciones del motor de archivos de comandos de Netscape, invocar la función alert() desde un controlador de eventos SVG podría no funcionar y provocar que no se ejecuten los JavaScripts posteriores. Problemas similares se producen en cualquier ventana modal de Netscape, incluida la ventana de privilegios de seguridad. Este error parece haberse corregido parcialmente en Netscape 4.75, pero no en Windows 98.
  8. Hay un error en Netscape relacionado con plug-ins que utilizan JavaScript. Si carga una página con un plug-in que utiliza JavaScript y, a continuación, desactiva Java y sale del explorador, Netscape quedará bloqueado.
  9. Netscape no siempre le permite acceder a la matriz document.embeds[] desde un archivo de comandos SVG. Para solucionar esto, utilice document.mySVG, donde mySVG es el nombre del objeto incrustado.
  10. Los errores en archivos de comandos ejecutados por plug-ins de Netscape 4.x no se muestran en la ventana de la consola Javascript.
  11. Netscape 4.x pierde una pequeña cantidad de memoria cada vez que un plug-in ejecuta un archivo de comandos.
  12. Netscape 4.x se bloqueará al cerrarse si se inició haciendo doble clic para cargar una página Web con contenido de plug-in LiveConnect.
  13. Netscape se bloqueará si ejecuta el comando "Vista preliminar" en una página que contenga plug-ins (como el Visor SVG de Adobe). Por una razón similar, Netscape también se bloquea al intentar imprimir una página que contenga plug-ins que utilicen Acrobat Distiller.
  14. Si utiliza Netscape 4.0x para ver el mismo archivo SVG en dos ventanas, y dicho archivo utiliza una función JavaScript global en el objeto de ventana, JavaScript no se inicializará en ocasiones en la segunda ventana. La solución es actualizar a una versión más reciente de Netscape.
  15. Debido a un error en Netscape 6, hacer clic con el botón derecho del ratón dentro del área SVG para abrir un menú contextual falla en Netscape 6. En su lugar, se abre el menú contextual de Netscape 6.

Historial de revisiones

Cambios desde la versión 1.0 del Visor SVG de Adobe

  • Elementos y atributos:
    • Compatibilidad agregada para los siguientes elementos: altGlyph, altGlyphDef, font, font-face, glyph, glyphRef, hkern, missing-glyph, mpath, pattern y vkern.
    • Compatibilidad agregada para los atributos de presentación de SVG
    • Compatibilidad terminada para el uso del elemento use de elementos symbol.
    • Compatibilidad agregada para el atributo spreadMethod en degradados.
    • Compatibilidad agregada para el nuevo atributo primitiveUnits en efectos de filtro.
    • Compatibilidad agregada para el modo "discreto" de feComponentTransfer.
  • Propiedades CSS:
    • Compatibilidad agregada para las siguientes propiedades CSS: alignment-baseline, color-interpolation, direction, dominant-baseline, flood-color, flood-opacity, kerning, lighting-color, pointer-events y unicode-bidi. La compatibilidad con la propiedad pointer-events no incluye la transparencia de elementos de trama.
    • Compatibilidad agregada para la propiedad font-variant, pero sólo cuando se aplica a las fuentes SVG.
    • Compatibilidad eliminada para la propiedad baseline-identifier, que se ha sustituido por la propiedad alignment-baseline en la última especificación aceptada.
    • Han cambiado los valores predeterminados de las propiedades clip-rule y fill-rule de even-odd a non-zero para reflejar los cambios en la última especificación aceptada.
    • La última especificación preliminar de la Recomendación posible para SVG cambia la forma en que se manejan las unidades CSS. Ahora se convierten al espacio del usuario al analizar, en lugar de en el momento de la visualización. Esto cambia notablemente la forma en que funciona el comando "Zoom".
  • DOM:
    • La compatibilidad con el Nivel 2 de DOM del Visor se ajusta ahora a la especificación de la revisión posible de DOM 2, a excepción de la interfaz keyEvent, que no se define en dicha especificación. La compatibilidad con la interfaz keyEvent se ajusta a la última versión publicada de DOM 2 de esa interfaz, en la especificación preliminar del 9/99.
    • Compatibilidad agregada para la interfaz SVGRect.
    • Compatibilidad agregada para la interfaz SVGPoint, a excepción del método matrixTransform.
    • Compatibilidad agregada para la interfaz SVGTextContentElement, a excepción de las propiedades getTextLength y getLengthAdjust.
    • Compatibilidad agregada para rootElement, getRootElement, currentScale, getCurrentScale, setCurrentScale, currentTranslate, getCurrentTranslate y setCurrentTranslate de la interfaz SVGDocument.
    • Compatibilidad agregada para los métodos DOM2 hasAttribute y hasAttributeNS de la interfaz Element.
    • Compatibilidad agregada para un nuevo método no estándar en el objeto Document llamado garbageCollect. Este método no lleva ningún parámetro y no devuelve ningún resultado. Invocar este método liberará cualquier objeto Node previamente eliminado del árbol. Si un archivo de comandos ha realizado una operación que da lugar a la eliminación de un nodo del árbol, una vez que el archivo de comandos ya no tiene uso para el nodo eliminado (y devuelto), el archivo de comandos debería invocar garbageCollect en el documento SVG. Una vez invocado, cualquier referencia de un archivo de comandos a un nodo que no era parte del documento cuando se invocó garbageCollect siempre fallará al acceder a su método. Esta es la única ventaja de los exploradores Netscape, ya que LiveConnect de Netscape no ofrece compatibilidad con referencias débiles. Para obtener una fácil compatibilidad multiplataforma, esta llamada se puede hacer bajo Internet Explorer, pero no servirá de nada en esa plataforma.
  • Otros cambios:
    • Para cumplir con la última especificación preliminar de la Recomendación posible para SVG, se ha eliminado la compatibilidad para las unidades userSpace.
    • Compatibilidad agregada para hojas de estilo de usuario. El Visor SVG busca un archivo llamado SVG-User-Styles.css, de acuerdo con la siguiente estrategia de búsqueda:

      Sistema operativoRutas buscadas
      Windows 2000
      1. Mis documentos
      2. \Documentos y configuración\<usuario>\Datos de programa
      3. \Documentos y configuración\<usuario>\Configuración local\Datos de programa
      4. \Documentos y configuración\Todos los usuarios\Datos de programa
      Windows 98
      1. Mis documentos
      Windows NT
      1. \WINNT\Perfiles\<usuario>\Personal
      2. \WINNT\Perfiles\<usuario>\Datos de programa
      Macintosh OS 8.x
      1. Carpeta del Sistema:Preferencias
      2. Volumen de inicio:Documentos
      Macintosh OS 9.x
      (único usuario)
      1. Carpeta del Sistema:Preferencias
      2. Carpeta del Sistema:Usuarios:<usuario>
      3. Volumen de inicio:Documentos
      Macintosh OS 9.x
      (multiusuario)
      1. Volumen de inicio:Usuarios:<usuario>:Preferencias
      2. Volumen de inicio:Documentos
    • El Visor SVG de Adobe lleva ahora incorporada su propia compatibilidad de administración del color. Esta compatibilidad intentará localizar un perfil de dispositivo adecuado para su pantalla. Si su ordenador tiene instalado software de administración del color y se ha seleccionado un perfil de monitor, el Visor utilizará también este perfil de monitor. Si no tiene instalada administración del color, o la ha desactivado, el Visor buscará un perfil de configuración de monitor de Adobe en el sistema. Este perfil lo crea la herramienta Adobe Gamma (instalada con otros productos Adobe) y se almacena generalmente en los mismos directorios que otros perfiles de color del sistema host. Si no se encuentra ningún perfil de dispositivo adecuado, el Visor recurrirá al perfil sRGB estándar y lo utilizará para todas las interpretaciones.
    • El plug-in de Netscape admite ahora acceso a DOM desde clientes Java.
    • El control ActiveX admite ahora controladores de eventos que no son de JavaScript (p. ej. Visual Basic).
    • Modificaciones para la compatibilidad con Windows 2000.
    • Un método alternativo agregado para panoramizar gráficos SVG con el teclado como medio de acceso. Si el bloqueo de desplazamiento está activado, las teclas de flecha panoramizarán los gráficos SVG que tienen el enfoque.
    • Las mejoras en el rendimiento implican múltiples animaciones simultáneas dentro del mismo archivo SVG. Los archivos SVG con animaciones simultáneas pequeñas, ampliamente espaciadas, se deben mostrar por lo general más rápido en la versión actual que en la Versión 1.0.
    • Mejoras en el rendimiento del plug-in de Mac.
    • Compatibilidad mejorada para texto vertical.
    • Compatibilidad agregada para texto bidireccional.
    • Optimizaciones en el rendimiento general.
    • feImage modificado para funcionar como el elemento use.
    • Compatibilidad actualizada para imágenes PNG.
    • Optimizaciones en el uso de memoria.
    • Compatibilidad agregada para rutas de animaciones.
